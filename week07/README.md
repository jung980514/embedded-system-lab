## 2.실험 이론
### 인터럽트

&nbsp;&nbsp;인터럽트는 하드웨어의 변화를 감지해 외부로부터 전기신호 입력을 CPU가 알아채는 방법이다. 인터럽트가 발생할 때마다 컨트롤러는 현재 명령의 실행을 완료(또는 잠시 멈추고)하고 인터럽트 서비스 루틴(ISR) 또는 인터럽트 핸들러의 실행을 시작한다.
</br>&nbsp;&nbsp;ISR은 프로세서나 컨트롤러에 인터럽트가 발생할 때 무엇을 해야 하는지 알려준다. 인터럽트는 하드웨어 인터럽트 또는 소프트웨어 인터럽트가 될 수 있다.

### 폴링

&nbsp;&nbsp;폴링은 Hardware의 변화를 지속적으로 읽어들여 변화를 알아채는 방법이다. 신호를 판단하기 위해 지속적으로 확인해야 한다 다른 일을 하는 중에 신호를 읽을 수 없다
